# Running Guide (Linux)

This repository contains **multiple sub-projects** (Python services, static 3D UI, React dashboards, RL training scripts, and notebooks). This guide shows how to run each piece.

> Tip: Don’t try to run everything at once on the same ports. In particular, both the static 3D UI and the RL frontend default to port `3000`.

---

## 0) Prerequisites

- **Python**: 3.8+ recommended (`python3 --version`)
- **Node.js**: 18+ recommended (`node -v`, `npm -v`)
- **Git** (optional)

---

## 1) 3D Simulation UI (static website)

Folder: `3D-Simulation/`

This is the Three.js-based simulation UI (`index.html` + `index.js`). It can be served with Python’s built-in HTTP server.

### Run

```bash
cd 3D-Simulation
python3 -m http.server 3000
```

Then open:
- `http://localhost:3000/index.html`

---

## 2) Multi-Drone WebSocket Server + Clients + Admin Dashboard

Folder: `3D-Simulation/`

This is the distributed multi-drone demo:
- Server: `multi_drone_server.py` (WebSocket on port `8765`)
- Drone clients: `multi_drone_client.html`
- Admin dashboard: `admin_dashboard.html`

### 2.1 Install Python deps (server)

```bash
cd 3D-Simulation
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements_multi_drone.txt
```

### 2.2 Start the WebSocket server

```bash
cd 3D-Simulation
source .venv/bin/activate
python3 multi_drone_server.py
```

Server listens on:
- `ws://0.0.0.0:8765`

### 2.3 Serve the HTML files (in a second terminal)

```bash
cd 3D-Simulation
python3 -m http.server 3000
```

### 2.4 Open clients

On the **same machine** (quick local test):
- Drone D1: `http://localhost:3000/multi_drone_client.html?drone_id=D1&server=localhost`
- Drone D2: `http://localhost:3000/multi_drone_client.html?drone_id=D2&server=localhost`
- Drone D3: `http://localhost:3000/multi_drone_client.html?drone_id=D3&server=localhost`
- Admin: `http://localhost:3000/admin_dashboard.html?server=localhost`

On **multiple laptops** on the same Wi‑Fi/LAN:
1. Run the server on the “admin” laptop.
2. Find its IP (example): `192.168.1.100`
3. Open clients on other laptops by setting `server=<admin-ip>`:

Example:
- `http://<client-laptop>:3000/multi_drone_client.html?drone_id=D1&server=192.168.1.100`

Notes:
- Ensure firewall allows inbound TCP `8765` on the server machine.

### 2.5 Assisted setup script (optional)

There is a helper script that prints the right URLs and can try to open the admin dashboard:

```bash
cd 3D-Simulation
source .venv/bin/activate
python3 setup_multi_drone.py
```

## 3) Email Notification System (human detection)

Folder: `3D-Simulation/`

The WebSocket multi-drone server imports `human_detection_email.py` and attempts to send an email whenever a human is detected.

### Configure email credentials

Edit the config inside `3D-Simulation/human_detection_email.py`:
- `sender_email`
- `sender_password` (use a Gmail **App Password** if using Gmail)
- `recipient_email`

### Run

Start the server (same as Section 2.2):

```bash
cd 3D-Simulation
source .venv/bin/activate
python3 multi_drone_server.py
```

Then run drone clients and trigger detections.

Known note:
- `3D-Simulation/README_EMAIL_SYSTEM.md` mentions a `test_email_system.py`, but that file is **not present** in this workspace. Use `multi_drone_server.py` to exercise the email flow.

## 6) RL Training Frontend Report (React / CRA)

Folder: `RL_Training_FrontendReport/`

This is a Create-React-App project (`react-scripts`).

### Install + run

```bash
cd RL_Training_FrontendReport
npm install
npm start
```

Default URL:
- `http://localhost:3000`

Port conflict note:
- If you are already using `python3 -m http.server 3000` for the 3D UI, run CRA on a different port:

```bash
PORT=3001 npm start
```

Then open:
- `http://localhost:3001`

## 8) (Optional / Advanced) `3D-Simulation/backend.py`

`3D-Simulation/backend.py` is a separate WebSocket server that appears to stream **simulated** detections using:
- OpenCV (`cv2`)
- PyTorch (`torch`, `torchvision`, `torchaudio`)
- Ultralytics YOLO (`ultralytics`)

Important:
- It expects model weights under `3D-Simulation/models/` (e.g. `models/image.pt`, `models/thermal.pt`, `models/screaming_detector_gpu.pth`), but **this workspace currently has no `models/` directory**.

If you add the required weights and want to try running it anyway:

```bash
cd 3D-Simulation
python3 -m venv .venv-backend
source .venv-backend/bin/activate
pip install --upgrade pip
pip install websockets opencv-python torch torchvision torchaudio ultralytics
python3 backend.py
```

## Troubleshooting

- **Port already in use**: change ports (e.g. `python3 -m http.server 3001`, or `PORT=3001 npm start`).
- **Multi-laptop setup doesn’t connect**:
  - confirm same Wi‑Fi/LAN
  - allow firewall inbound `8765/tcp` on the server machine
  - make sure client URLs use `server=<server-ip>`
- **Email doesn’t send**:
  - update credentials in `3D-Simulation/human_detection_email.py`
  - for Gmail, use an **App Password** and ensure 2FA is enabled
